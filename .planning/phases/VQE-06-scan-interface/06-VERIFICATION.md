# Phase 6: Scan Interface & API - Verification

## Release Criteria
- [ ] `/api/scan` endpoint securely handles token validation.
- [ ] Atomic DB updates prevent race conditions for double-scanning.
- [ ] Guard interface prompts for camera access and scans QRs reliably.
- [ ] UI provides distinct visual feedback for Success, Duplicate, and Invalid states.

## Verification Steps
1. Navigate to `/scan` (log in as a Guard or Admin).
2. Allow camera permissions.
3. Present a generated QR code from Phase 4 to the camera.
4. Verify the screen flashes green and says "Success".
5. Present the exact same QR code again.
6. Verify the screen flashes yellow/red and says "Already Scanned".
7. Present a random QR code (not generated by the system).
8. Verify the screen flashes red and says "Invalid Token".
